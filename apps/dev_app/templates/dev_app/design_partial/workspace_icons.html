Color Adaptation

Icons automatically adapt to workspace color schemes using CSS variables.

Text Colors
/* Use workspace icon colors */
.workspace-icon {
  color: var(--workspace-icon-primary);
}

.workspace-icon-secondary {
  color: var(--workspace-icon-secondary);
}

.workspace-icon-muted {
  color: var(--workspace-icon-muted);
}
State Colors
/* Interactive states */
.workspace-icon:hover {
  color: var(--scitex-blue);
  opacity: 1;
}

.workspace-icon.active {
  color: var(--scitex-blue);
}

Best Practices

  • ✓ Use semantic icons - Choose icons that clearly represent their action or content
  • ✓ Maintain consistent sizing - Use standard sizes (16px, 20px, 24px, 32px) for visual hierarchy
  • ✓ Use workspace color variables - Icons should adapt to light/dark themes automatically
  • ✓ Provide hover states - Interactive icons should have clear hover feedback
  • ✗ Don't mix icon styles - Use consistent FontAwesome style (solid, regular, brands)
  • ✗ Don't use icons without context - Always provide tooltips or labels for clarity
