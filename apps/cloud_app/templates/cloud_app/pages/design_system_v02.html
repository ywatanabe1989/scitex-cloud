Usage Guidelines

The following guidelines ensure consistent implementation of the design system:

‚úÖ Do:

  • Consistency: Use same components across all pages
  • Color Harmony: Stick to the defined SciTeX palette
  • Spacing: Use the 0.25rem increment spacing scale
  • Typography: Follow the established font hierarchy
  • Accessibility: Ensure WCAG AA compliance
  • Responsiveness: Design for mobile-first approach
  • Performance: Use lightweight CSS with minimal dependencies
  • Testing: Test components across browsers and devices

‚ùå Don't:

  • Custom Colors: Create colors outside the SciTeX palette
  • Inconsistent Spacing: Use arbitrary spacing values
  • Bootstrap Overrides: Override Bootstrap without SciTeX alternatives
  • Typography Mix: Use fonts not specified in the design system
  • Accessibility Issues: Ignore color contrast and keyboard navigation
  • Responsive Neglect: Design only for desktop
  • Undocumented Variants: Create custom component variants without documentation
  • Color Dependencies: Rely on color alone to convey information

Hero Gradient Implementation Guide

Current Usage:

  • Landing Page: .hero-silverish-ai - Main platform introduction with 4-color gradient
  • Scholar Search: .search-hero - Simple 2-color gradient for focused search interface
  • Module Pages: Variations of 2-color gradients for specific applications

Implementation Pattern:

  • Primary Landing: Rich 4-stop gradient for maximum visual impact
  • Application Pages: Simplified 2-stop gradients for better content focus
  • Search Interfaces: Dark-to-medium gradients with white overlay areas

CSS File Location:

Design Principles:

  • Gradient Direction: Consistent 135deg diagonal for visual flow
  • Color Progression: Dark to light creates depth and dimension
  • Text Contrast: Text shadows ensure readability across gradient transitions
  • Brand Colors: All gradients use SciTeX color palette exclusively
  • Responsive Design: Gradients adapt to all screen sizes

Best Practices:

  • Content Overlay: Use semi-transparent cards over gradients for better readability
  • Text Shadows: Apply appropriate text shadows for contrast
  • Color Consistency: Stick to established gradient patterns
  • Performance: CSS gradients are lightweight and performant
