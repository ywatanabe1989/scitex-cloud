How to Contribute

Getting Started

New to contributing to open source? Follow these steps to make your first contribution to SciTeX:

  1. Find a repository that interests you from the list above
  2. Fork the repository to your GitHub account
  3. Clone your fork to your local machine
  4. Create a new branch for your changes
  5. Make your changes following our coding guidelines
  6. Push your changes and create a pull request

Our maintainers will review your contribution and provide feedback.

Ways to Contribute

  • Code Contributions - Fix bugs, implement new features, or improve existing ones
  • Documentation - Improve or expand documentation, tutorials, and examples
  • Bug Reports - Help identify and document issues
  • Feature Requests - Suggest new features or improvements
  • Testing - Help test new features and ensure quality
  • Design - Contribute improved UI/UX designs

Contribution Guidelines

  • Follow the code style of the specific repository
  • Write meaningful commit messages
  • Document your changes thoroughly
  • Add tests for new functionality
  • Ensure all tests pass before submitting
  • Be respectful and constructive in discussions
