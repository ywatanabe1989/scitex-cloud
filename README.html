Quick Start

Docker (Recommended) 
# Clone and navigate
git clone git@github.com:ywatanabe1989/scitex-cloud.git
cd scitex-cloud

# Start development environment
make start

# Access at: http://localhost:8000
# Gitea: http://localhost:3000

Test User: - Username: test-user - Password: Test-user!

Local (Without Docker) 
# Setup
python3 -m venv .venv
source .venv/bin/activate
pip install scitex[web,scholar,writer,dev]

# Configure
cp deployment/dotenvs/dotenv.example .env
# Edit .env with your settings

# Run
python manage.py migrate
python manage.py runserver

# Access at: http://127.0.0.1:8000

Deployment Options

Development (Default) 
make start                    # Full setup
make restart                  # Quick restart
make logs                     # View logs
make migrate                  # Run migrations
make shell                    # Django shell
Production 
make ENV=prod start           # Deploy production
make ENV=prod db-backup       # Backup database
make ENV=prod verify-health   # Health check
NAS/Home Server 
make ENV=nas start            # Start on NAS
make ENV=nas status           # Check status
make ENV=nas db-backup        # Backup

All commands: make help or make ENV=prod help

Environment Variables

Configuration Files

Place .env files in SECRET/ directory (gitignored): - SECRET/.env.dev - Development - SECRET/.env.prod - Production - SECRET/.env.nas - NAS

Required variables:

# Django
DJANGO_SECRET_KEY=your-secret-key
DEBUG=True                           # False in production

# Database
POSTGRES_DB=scitex_cloud_dev
POSTGRES_USER=scitex_dev
POSTGRES_PASSWORD=strong-password

# Gitea (optional)
SCITEX_CLOUD_GITEA_URL=http://gitea:3000
SCITEX_CLOUD_GITEA_TOKEN=your-token

Templates available: - containers/docker_dev/.env.dev.example - containers/docker_prod/.env.prod.example - containers/docker_nas/.env.nas.example

Architecture

Tech Stack

Backend: - Django 4.2+ - PostgreSQL (Docker) / SQLite (local) - Gunicorn (production)

Frontend: - HTML5, CSS3, JavaScript - Theme-responsive (light/dark modes) - GitHub-inspired UI

Infrastructure: - Nginx (reverse proxy) - Gitea (Git hosting) - Redis (caching) - Docker Compose (orchestration)

Design: - Project-centric (all modules link to projects) - 100% MIT licensed
